## Project Overview

This is a personal website (joshuami.com) built with **Drupal 11** and deployed as **static HTML to GitHub Pages** using the Tome module. The site has evolved through three platforms: WordPress → Jekyll → Drupal (with static generation).

## Project Structure

```
/
├── web/                    # Drupal application root
│   ├── modules/            # Custom and contrib modules
│   ├── themes/custom/xtra/ # Custom Drupal theme
│   └── presentations/      # Reveal.js presentations
├── html/                   # Generated static HTML (deployed to GitHub Pages)
├── content/                # Drupal content files (Tome content storage)
├── config/                 # Drupal configuration exports
├── vendor/                 # PHP dependencies (Composer)
├── node_modules/           # Node.js dependencies
├── .ddev/                  # DDEV local development configuration
├── recipes/                # Drupal recipes for installation profiles
└── patches/                # Custom patches for contrib modules
```

## Static HTML Deployment to GitHub Pages

### How It Works

1. **Content is managed in Drupal** running locally via DDEV
2. **Tome module generates static HTML** to the `/html` directory
3. **GitHub Actions deploys `/html`** to GitHub Pages on push to `main`

### Build Process

Run the static build with Composer:

```bash
composer static
```

This executes:
1. Creates a database backup in `/backups`
2. Runs `drush tome:static` to generate static HTML
3. Copies presentations from `web/presentations` to `html/presentations`
4. Copies original images to the static output

### GitHub Actions Workflow

The workflow in `.github/workflows/static.yml`:
- Triggers on push to `main` branch or manual dispatch
- Uploads the `/html` directory as an artifact
- Deploys to GitHub Pages using `actions/deploy-pages@v4`

### Key Commands

```bash
# Full static build
composer static

# Just run tome export
ddev drush tome:static -l --uri=https://joshuami.com

# Copy presentations manually
composer copy-presentations

# Start local development
ddev start
```

## Creating Presentations

Presentations use **Reveal.js 4.6.1** with a custom theme and Vite for bundling. See `web/presentations/README.md` for comprehensive documentation and `web/presentations/BUILD.md` for build details.

### Quick Start

1. **Create a new presentation**:
   ```bash
   mkdir web/presentations/your-presentation-name
   cp web/presentations/template.html web/presentations/your-presentation-name/index.html
   ```

2. **Edit the presentation** (`index.html`):
   - Update the `<title>` tag
   - Set a background image for the title slide
   - Add slides inside `<div class="slides">` (each `<section>` is a slide)

3. **Preview locally**:
   ```bash
   npm run dev    # Opens at http://localhost:8080/template.html
   ```

4. **Deploy**: Run `composer static` to include in the static build

### Basic Slide Structure

```html
<!-- Title slide with background image -->
<section class="title-slide" style="background-image: url('path/to/image.jpg') !important;">
  <div class="title-content">
    <h1>Presentation Title</h1>
    <p>Subtitle or author info</p>
  </div>
</section>

<!-- Regular slide -->
<section>
  <h2>Slide Title</h2>
  <p>Content here</p>
</section>

<!-- Vertical slides (nested sections) -->
<section>
  <section><h2>Main Topic</h2></section>
  <section><h3>Sub-topic 1</h3></section>
  <section><h3>Sub-topic 2</h3></section>
</section>
```

### Layout Classes

- `slide-layout-two-column` - Two-column grid layout
- `slide-layout-three-column` - Three-column grid layout
- `text-center`, `text-left`, `text-right` - Text alignment
- `small`, `large` - Text sizing
- `fragment` - Progressive disclosure (appears on click)

## Notes

- The `.nojekyll` file in `/html` disables Jekyll processing on GitHub Pages
- The `CNAME` file sets the custom domain to joshuami.com
- Database backups are stored in `/backups` (not committed)
- The editoria11y module has a custom patch for update 9011
